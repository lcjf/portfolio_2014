    • Making life easier with Compass and SASS

    A brief example of how you can utilise this CSS Authoring Framework to your advantage.

    Follow this installation guide to get up and running, and then modify your config.rb file to correctly reference your folder names.

    More info on the below can be found using the documentation.

    Gradients

    This will generate all the necessary vendor prefixes.

    Example
    @include background-image(linear-gradient($mine-shaft, $mine-shaft - 100));

    Border Radius

    Same as above; this will generate all the necessary vendor prefixes.

    Example
    @include border-radius(5px);

    Transitions

    The below example fades in all properties, using the 'ease-in' transition, over a time of 200ms.

    Example
    @include transition-property(all);
@include transition-duration(.2s);
@include transition-timing-function(ease-in);
&:hover {
    color: $mine-shaft - 100;
    background: darken($golden-rod, 15%);
}

    This also works using the shorthand method.

    @include transition(all .2s ease-in);

    Sprites

    Another brilliant feature of Compass. This will automatically generate both your image sprite (based on a folder of images) and CSS.

    Create a sub-folder inside your images folder with a sensible name, copy all your images in to it, then add the below line of code to your main stylesheet where you have declared all your imports (typically style.scss, stylesheet.scss or application.scss).

    @import "your_folder_name/*.png";

    You can also use @include to define select images from your sprite.

    .name_of_image {
    @include your_folder_name(name_of_image);
}

    Advanced Sprites

    This example demonstrates how the output file can be further customised.

    $icons-spacing: 20px;
$icons-flickr-spacing: 80px;
$icons-layout: horizontal;
$icons-layout: diagonal;
$icons-layout: smart;

    Hover states

    If you want to create a hover state for one of your sprites, all you have to do is create another image with the same name, but append '_hover' to the end e.g. 'facebook_hover.png'.

    Horizontal and vertical lists

    This will automatically generate the styles required for your lists and their child elements.

    .lists ul {
    @include horizontal-list;
    a {
        display: block;
        width: 48px;
        height: 48px;
    }
}

    Vertical Rythm

    On the Web, vertical rhythm – the spacing and arrangement of text as the reader descends the page – is contributed to by three factors: font size, line height and margin or padding. All of these factors must be calculated with care in order that the rhythm is maintained.

    Source: http://24ways.org/2006/compose-to-a-vertical-rhythm

    The mixin and variables must be set for you to establish your vertical rhythms.

    @include establish-baseline;
$base-font-size: 16px;
$base-line-height: 24px;

    These 2 examples would typically be included on the opening paragraph and closing paragraphs of your document. You could include these by utlising pseudo selectors, such as first-child, last-child, first-of-type and last-of-type:

    .paragraph-one {
    @include leader();
}
.paragraph-four {
    @include trailer();
}

    If using vertical rhythms then you must change your font sizes using adjust-font-size, this is so that all values are re-calculated accordingly.

    .paragraph-three {
    @include adjust-font-size-to(20px);
}

    This is how to further customise elements, whilst still maintaining your vertical rhythm

    .paragraph-two {
    $default-rhythm-border-style: dashed;
    @include rhythm-borders(2px);
}
